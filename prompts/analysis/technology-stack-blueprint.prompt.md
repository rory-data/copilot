# Technology Stack Blueprint Generator

## Goal

Generate comprehensive technology stack blueprints following **Core Engineering Principles** and **Workflow Standards** for consistent development guidance.

## Quick Reference

- **Analysis Framework**: Systematic technology identification and documentation
- **Engineering Focus**: SOLID principles applied to architecture analysis
- **Quality Standards**: Maintainable, scalable technology choices
- **Documentation**: Clear, actionable technology blueprints

## Configuration

**Analysis Depth**: Basic | Standard | Comprehensive | Implementation-Ready  
**Technology Focus**: Auto-detect | .NET | Java | JavaScript | React | Python | Other  
**Include Details**: Versions | Licenses | Diagrams | Usage Patterns | Conventions  
**Output Format**: Markdown | JSON | YAML | HTML

## Analysis Framework

### Phase 1: Technology Discovery

1. **Project Structure Analysis**: Scan configuration files and project structure
2. **Dependency Extraction**: Parse package.json, .csproj, pom.xml, requirements.txt
3. **Build Tool Detection**: Identify build scripts, CI/CD pipelines
4. **Version Mapping**: Extract precise version information

### Phase 2: Architecture Analysis

1. **Layer Identification**: Presentation, business logic, data access
2. **Pattern Recognition**: Design patterns, architectural styles
3. **Integration Points**: APIs, databases, external services
4. **Quality Attributes**: Performance, security, scalability approaches

### Phase 3: Documentation Generation

1. **Technology Categorization**: Group by type, layer, or purpose
2. **Usage Pattern Documentation**: How technologies are implemented
3. **Convention Extraction**: Coding standards and practices
4. **Blueprint Creation**: Implementation-ready documentation

### JavaScript/Node.js Stack

- Runtime versions and package managers
- Frontend frameworks and libraries
- Build tools and bundlers
- Testing frameworks

### Python Stack

- Python version and virtual environment
- Package dependencies and versions

- Data processing libraries
- Testing frameworks
- Code quality tools

- **SOLID Principles**: Architecture must support maintainable design
- **Clean Code**: Technology choices support readable, maintainable code
